# Tshuabu 开发计划（MVP优先：本地可追溯 + Image API桥接）

更新时间：2026-05-15

## 1. 目标与边界

你的核心目标不是先做完整画布，而是先做一个**稳定可用的 image 工作台核心闭环**：

1. 调用 image API（text2img / img2img / inpaint）
2. 每次操作都记录到本地（可作为 RAG 材料）
3. 需要时可以快速检索、阅读、复盘

> 结论：先做“可记录、可复盘、可继续开发”的底座，再扩展 UI。

---

## 2. 版本路线

### V0（1~2天）——先把记录系统做通

- [ ] 统一接口调用层（至少 text2img + inpaint）
- [ ] 本地日志双写：
  - `RAG_LOG.md`（人类阅读）
  - `rag_events.jsonl`（机器检索）
- [ ] 每次请求记录：输入参数、输出图链接、耗时、错误
- [ ] 能按会话查看最近记录

产出标准：
- 你能在本地看到完整操作轨迹
- 任何一次出图都能回溯“当时参数 + 结果”

### V1（第1周）——补齐最小可用产品体验

- [ ] 加上 img2img 接口
- [ ] 提示词按钮模板（本地保存、点击插入）
- [ ] 日志检索命令（按关键词/日期/模型）
- [ ] 失败重试与错误分类

产出标准：
- 你不用反复手打常用提示词
- 能快速找回“哪次参数最好”

### V2（第2~4周）——趋近成熟工具

- [ ] 项目维度归档（按项目/任务分组）
- [ ] 自动生成 session 摘要（供RAG）
- [ ] 供应商适配层（多模型切换）
- [ ] 成本统计（每次调用耗时/费用）

## 4. 日志字段规范（关键）

每条事件建议包含：

- `event_id`：唯一ID
- `timestamp`：ISO时间
- `session_id`：一次连续创作会话ID
- `action`：`text2img | img2img | inpaint | prompt_template_use | prompt_template_create`
- `model`
- `prompt`
- `negative_prompt`
- `params`：`size, steps, strength, seed...`
- `input_assets`：原图/mask路径或URL
- `output_assets`：结果图路径或URL
- `status`：`success | failed`
- `latency_ms`
- `error_message`（失败时）

## 5. RAG材料落地策略（不依赖画布）

### 5.1 `rag_events.jsonl`（机器）
- 一行一个 JSON 事件，便于后续 embedding / 检索。

### 5.2 `RAG_LOG.md`（人）
- 每次事件追加一段可读摘要：
  - 做了什么
  - 用了什么 prompt 与参数
  - 结果如何
  - 下一步建议

示例（MD）：

```md
## 2026-05-15 14:31 inpaint success
- session: s_20260515_a1
- model: xxx
- prompt: "保留人物脸部，增强衣服纹理，电影感光影"
- input: outputs/base_001.png + outputs/mask_001.png
- output: outputs/inpaint_001.png
- latency: 8421ms
- note: 纹理增强明显，但高光略过曝，下次 strength 降到 0.45
```

## 6. 两天执行清单（可直接照做）

### Day 1

1. 建目录 + 日志文件初始化
2. 打通 text2img API
3. 每次调用写入 `jsonl + md`
4. CLI 命令：`run text2img`、`show latest logs`

验收：
- 至少成功生成3张图，且都能在日志追溯

### Day 2

1. 打通 inpaint（原图 + mask + prompt）
2. 增加提示词模板（保存/调用）
3. 查询命令（按关键词、按日期）
4. 失败场景验证（超时/空响应）

验收：
- 完成 1 次“生成→修改prompt→局部重绘→复盘日志”闭环

---

## 7. 风险与规避

1. **供应商参数不统一**
   - 规避：在中转层统一字段（内部协议），各家做 adapter
2. **日志不完整导致 RAG 无用**
   - 规避：失败事件也必须写日志
3. **路径混乱**
   - 规避：统一 outputs 命名规则（日期+序号）

---

## 8. 当前优先级建议

先做：
1. API桥接
2. 本地日志双写（md + jsonl）
3. 提示词模板

后做：
1. 画布可视化
2. 多人协作
3. 完整产品化能力

> 这条路线能确保你很快有“可用、可积累、可进化”的基础系统。
