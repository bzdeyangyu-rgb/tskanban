# 第一阶段开发反馈待处理池

更新时间：2026-05-29

## 使用规则

本文件用于记录用户在本轮开发过程中发现的问题。

本轮规则：

1. 用户反馈的问题先记录，不立即修改。
2. 除非用户明确说“这是阻塞，马上停下来修”，否则继续按当前开发计划推进。
3. 当前开发进度完成后，统一整理所有反馈，形成第二版修改计划。
4. 第二版修改计划要把反馈问题重新归类、排序、合并，避免边开发边跑偏。

## 分类标准

| 分类 | 说明 |
|---|---|
| UI 偏差 | 视觉、字号、颜色、布局、弹窗、按钮、节点外观 |
| 交互偏差 | 拖拽、框选、连线、断线、删除、resize、上传 |
| 功能偏差 | 生成、保存、恢复、循环、LLM、图片编辑、基因库 |
| 数据偏差 | 日志、版本、RAG、历史、持久化、API payload |
| 产品边界偏差 | MS、ComfyUI、重复入口、不要的功能重新出现 |
| 验收风险 | 测试通过但浏览器手感或样例对齐未确认 |
